1. はじめに

このガイドでは、タグラグビーAIをプログラミングする際に使用できるすべてのパラメータについて、 詳しく解説しています。各パラメータは、ゲームの状態を数値化したものであり、 これらの値を組み合わせることで、AIの行動を制御することができます。

パラメータは大きく分けて3つのカテゴリに分類されます： 共通変数（step, ball, select）、 移動評価パラメータ（移動の評価に使用）、 パス評価パラメータ（パスの評価に使用）です。 それぞれのパラメータは、移動評価のループまたはパス評価のループ内で自動的に計算され、 あなたのコードから直接使用することができます。

2. 共通で使用可能な変数

step
number（数値）

現在のステップ数（行動回数）を表す変数です。ゲームが開始されてから、何回目の行動かを数えています。 この値を使うことで、ゲームの進行状況に応じた戦略を実装できます。

値の範囲：0以上の整数
使用例：ゲーム序盤（step < 5）と中盤以降で異なる戦略を適用する
// ゲーム序盤は慎重に、中盤以降は積極的に if (step < 5) { // 序盤の戦略 move_score = -15 * back_forth_from_goalline; } else { // 中盤以降の戦略 move_score = -10 * back_forth_from_goalline; }
ball
number（数値）

ボールを持っているプレイヤーの番号を表します。プレイヤー番号は0から始まります。 この値を使うことで、ボール保持者かどうかを判定したり、ボール保持者に応じた戦略を実装できます。

値の範囲：0以上の整数（プレイヤー数-1まで）
使用例：現在のプレイヤー（select）がボールを持っているかを判定
// ボール保持者と非保持者で異なる評価を行う if (ball === select) { // ボール保持者の場合：前進を重視 move_score = 20 * back_forth_from_goalline; } else { // 非保持者の場合：ボール保持者に近づく move_score = -6 * vertical_diff_from_ball * vertical_diff_from_ball; }
select
number（数値）

現在行動するプレイヤーの番号を表します。プレイヤー番号は0から始まります。 この値を使うことで、特定のプレイヤーに応じた戦略を実装できます。

値の範囲：0以上の整数（プレイヤー数-1まで）
使用例：特定のプレイヤー（例：1人目）に特別な戦略を適用
// 1人目のプレイヤーに特別な戦略を適用 if (select === 0) { // 1人目の特別な評価 move_score = 10 * distance_defense_min; } else { // その他のプレイヤーの評価 move_score = 5 * distance_defense_min; }

3. 移動評価で使用可能なパラメータ

distance_defense
Array（配列）

現在の位置から、すべての守り手（ディフェンス）までの距離を配列で格納しています。 各要素は、対応する守り手との距離を表します。この配列を使うことで、 すべての守り手との距離を個別に評価できます。

値の範囲：各要素は0以上の数値（距離）
使用例：最も近い守り手との距離を取得する場合、distance_defense_minを使用することをお勧めします
distance_defense_min
number（数値）

すべての守り手の中で、最も近い守り手との距離を表します。 この値が大きいほど、守り手から離れている（安全）ことを意味します。 移動の評価において、最も重要なパラメータの一つです。

値の範囲：0以上の数値（距離が大きいほど安全）
推奨係数：5〜15程度（守り手から離れることを重視する場合）
// 守り手から離れることを重視する move_score = 10 * distance_defense_min; // 守り手から離れることを重視しつつ、前進も考慮 move_score = 8 * distance_defense_min + 5 * back_forth_from_goalline;
back_forth_from_goalline
number（数値）

ゴールラインに対する前後の移動距離を表します。 前進すると正の値、後退すると負の値、横移動は0になります。 この値を使うことで、前進を促したり、後退を促したりできます。

値の範囲：正の値（前進）、負の値（後退）、0（横移動）
推奨係数：-15〜-10（後退を重視）、10〜20（前進を重視）
// 後退を重視する（後退は+15、前進は-15） move_score = -15 * back_forth_from_goalline; // 前進を重視する（前進は+20、後退は-20） move_score = 20 * back_forth_from_goalline;

重要なポイント：係数に負の値を使うと後退を重視し、 正の値を使うと前進を重視します。例えば、-15 * back_forth_from_goalline の場合、 後退すると値がプラス（良い）、前進すると値がマイナス（悪い）になります。

horizontal_diff_from_ball
number（数値）

ボールを持っているプレイヤーとの横方向の距離（絶対値）を表します。 ボールとの横の距離が大きいほど、値が大きくなります。 この値を使うことで、ボール保持者と横に並ぶことを促したり、横に離れることを促したりできます。

値の範囲：0以上の数値（距離が大きいほど値が大きい）
使用例：ボール保持者と横に並ぶことを重視する場合、この値を小さくする
// ボール保持者と横に並ぶことを重視 move_score = -5 * horizontal_diff_from_ball;
vertical_diff_from_ball
number（数値）

ボールを持っているプレイヤーとの縦方向の距離を表します。 正の値はボールより前方、負の値はボールより後方を意味します。 この値を使うことで、ボール保持者と縦に並ぶことを促したり、前後に散らばることを促したりできます。

値の範囲：正の値（前方）、負の値（後方）
使用例：ボール保持者と同じ縦位置を保つことを重視する場合、この値の二乗を使う
// ボール保持者と同じ縦位置を保つことを重視（差の二乗で減点） move_score = -6 * vertical_diff_from_ball * vertical_diff_from_ball;
defenseLine
Array（配列）

守り手（ディフェンス）の左からの順番を表した配列です。 左から右に向かって、守り手のIDが格納されています。 この配列を使うことで、守り手の配置を分析できます。

値の範囲：配列の各要素はプレイヤーID（0以上の整数）
使用例：守り手の配置を分析する高度な戦略で使用
attackLine
Array（配列）

攻め手（アタック）の左からの順番を表した配列です。 左から右に向かって、攻め手のIDが格納されています。 この配列を使うことで、攻め手の配置を分析できます。

値の範囲：配列の各要素はプレイヤーID（0以上の整数）
使用例：攻め手の配置を分析する高度な戦略で使用
deviation_from_uniform_position
number（数値）

均等に横方向に並ぶ場合の場所からのずれを表します。 この値が小さいほど、均等に並んでいることを意味します。 この値を使うことで、攻め手が均等に横に並ぶことを促せます。

値の範囲：0以上の数値（値が小さいほど均等に近い）
使用例：均等に並ぶことを重視する場合、この値を小さくする
// 均等に並ぶことを重視 move_score = -3 * deviation_from_uniform_position;

4. パス評価で使用可能なパラメータ

distance_defense_throw_min
number（数値）

ボールを持っているプレイヤー（パスを出す側）とディフェンスとの最短距離を表します。 この値が大きいほど、パスを出す側が守り手から離れている（安全）ことを意味します。 パスを出す際の安全性を評価する重要なパラメータです。

値の範囲：0以上の数値（値が大きいほど安全）
推奨係数：4〜10程度（パスを出す側の安全性を重視する場合）
// パスを出す側が守り手から離れていることを重視 pass_score = 8 * distance_defense_throw_min;
pass_distance
number（数値）

パスの距離（ユークリッド距離）を表します。 パス距離が短いほど成功率が高いと考えられます。 この値が小さいほど、短いパスであることを意味します。

値の範囲：0以上の数値（値が小さいほど短いパス）
推奨係数：-3〜-5程度（短いパスを重視する場合）
// パス距離が短いことを重視（短いパスほど高評価） pass_score = -4 * pass_distance; // パス距離が短く、受け手が安全なことを重視 pass_score = -3 * pass_distance + 6 * distance_defense_catch_min;

重要なポイント：係数に負の値を使うと短いパスを重視し、 正の値を使うと長いパスを重視します。一般的には、短いパスの方が成功率が高いため、 負の係数を使うことが推奨されます。

distance_defense_catch_min
number（数値）

パスを受けるプレイヤーとディフェンスとの最短距離を表します。 この値が大きいほど、パスを受ける側が守り手から離れている（安全）ことを意味します。 パスを受ける際の安全性を評価する重要なパラメータです。

値の範囲：0以上の数値（値が大きいほど安全）
推奨係数：6〜10程度（パスを受ける側の安全性を重視する場合）
// パスを受ける側が守り手から離れていることを重視 pass_score = 10 * distance_defense_catch_min; // パスを受ける側の安全性と、パス距離のバランスを取る pass_score = 6 * distance_defense_catch_min - 3 * pass_distance;

5. 実践的な使用例

例1: ボール保持者と非保持者で異なる戦略

ボール保持者は前進を重視し、非保持者はボール保持者に近づく戦略の例です。

//移動の評価値計算 if (ball === select) { // ボール保持者は前進を重視 move_score = 20 * back_forth_from_goalline + // 前進を強く評価 4 * distance_defense_min; // 安全性も考慮 } else { // 非保持者はボール保持者に近づく move_score = 5 * distance_defense_min - // 守り手から離れる 6 * vertical_diff_from_ball * vertical_diff_from_ball; // ボール保持者と同じ縦位置 } //パスの評価値計算 // 安全で短いパスを重視 pass_score = 6 * distance_defense_catch_min - // 受け手が安全 3 * pass_distance + // パス距離が短い 4 * distance_defense_throw_min; // 出す側も安全

例2: ゲーム序盤と中盤以降で異なる戦略

ゲーム序盤は慎重に後退し、中盤以降は積極的に前進する戦略の例です。

//移動の評価値計算 if (step < 5) { // 序盤：慎重に後退 move_score = -15 * back_forth_from_goalline + // 後退を重視 8 * distance_defense_min; // 守り手から離れる } else { // 中盤以降：積極的に前進 move_score = 10 * back_forth_from_goalline + // 前進を重視 5 * distance_defense_min; // 安全性も考慮 } //パスの評価値計算 if (step < 5) { // 序盤：パスを出す側の安全性を重視 pass_score = 10 * distance_defense_throw_min; } else { // 中盤以降：短いパスを重視 pass_score = -4 * pass_distance; }

例3: 消極的で後退を重視する戦略

常に後退を重視し、守り手から離れることを優先する消極的な戦略の例です。

//移動の評価値計算 // 後退を重視し、守り手から離れることを優先 move_score = -12 * back_forth_from_goalline + // 後退を重視 8 * distance_defense_min; // 守り手から離れる //パスの評価値計算 // パスを出す側と受ける側の安全性を重視 pass_score = 6 * distance_defense_catch_min + // 受け手が安全 4 * distance_defense_throw_min; // 出す側も安全

6. ベストプラクティス

係数の調整について

パラメータに掛ける係数は、戦略の優先度を表します。係数が大きいほど、そのパラメータが重視されます。 ただし、係数を大きくしすぎると、他の要素が無視されてしまう可能性があるため、 バランスを取ることが重要です。

推奨される係数の範囲：
• 移動評価：各パラメータに5〜20程度の係数
• パス評価：各パラメータに3〜10程度の係数
• まずは小さな係数から始めて、徐々に調整していくことをお勧めします。

パラメータの組み合わせ

複数のパラメータを組み合わせることで、より洗練された戦略を実装できます。 例えば、前進を重視しつつ、守り手から離れることも考慮するなど、 複数の要素をバランスよく組み合わせることが重要です。

// 複数のパラメータを組み合わせた例 move_score = -10 * back_forth_from_goalline + // 後退を重視 8 * distance_defense_min + // 守り手から離れる -3 * deviation_from_uniform_position; // 均等に並ぶ

条件分岐の活用

step、ball、selectなどの変数を使った条件分岐により、 状況に応じた柔軟な戦略を実装できます。ゲームの進行状況やプレイヤーの状態に応じて、 異なる評価式を使い分けることができます。

条件分岐の例：
• ボール保持者かどうかで分岐（ball === select）
• ゲーム序盤かどうかで分岐（step < 5）
• 特定のプレイヤーかどうかで分岐（select === 0）
これらの条件分岐を組み合わせることで、より複雑な戦略を実装できます。

デバッグのヒント

コードにエラーがある場合、codeerror要素にエラーメッセージが表示されます。 エラーメッセージを確認して、問題を修正してください。 また、係数を調整する際は、少しずつ変更して、その効果を確認することをお勧めします。

よくあるエラー：
• 変数名のタイプミス（例：move_score と moveScore）
• 未定義の変数の使用
• 構文エラー（括弧の対応など）
これらのエラーは、エラーメッセージを確認することで、簡単に修正できます。
